# 近隣インシデントレポート 要件定義

## プロダクトコンセプト

> **「人を晒す地図」ではなく「地域の傾向を可視化する統計アプリ」**

- 個人を晒さない
- 「人」ではなく「出来事（インシデント）」を共有する
- 位置・時間はぼかす
- 通報導線を優先する

---

## 1. 画面設計

### ① マップ画面（Home）

**表示する情報**

- 100m〜200m グリッド単位のヒートマップ
- カテゴリフィルタ
- 期間フィルタ（過去 24h / 7日 / 30日）

**表示しない情報**

- ピン（メッシュ単位のみ、ピンは出さない）
- 正確な住所
- 投稿者情報
- リアルタイムデータ（最低 6〜12h 遅延）

---

### ② インシデント詳細画面

**表示する情報**

- カテゴリ
- 時間帯（例：22:00〜24:00）
- メッシュ位置（例：渋谷区○○ 100m グリッド）
- テキスト（200文字以内）
- 同メッシュ件数（統計）

**表示しない情報**

- 人物特徴
- 車のナンバー
- 顔写真
- 個人特定可能な情報

---

### ③ 投稿画面

**入力項目**

| 項目 | 必須 | 備考 |
|------|:----:|------|
| カテゴリ | ✅ | 騒音 / 危険行為 / ゴミ不法投棄 / 迷惑駐車 / その他 |
| 発生時間帯 | ✅ | 選択式（時間帯を選ぶ） |
| 位置 | ✅ | 自動メッシュ変換（生緯度は即時破棄） |
| 内容 | ✅ | 200文字制限、NGワードフィルタあり |
| 画像 | ❌ | 任意、自動モザイク処理必須 |

**制限事項**

- 正確な位置は保存しない（mesh_code のみ保存）
- 投稿直後は非公開（遅延公開）
- 同一ユーザーの連投制限（1日5件まで）

---

### ④ 通報導線画面

カテゴリに応じて適切な通報先を案内する。アプリは「晒す場」ではなく「相談の入口」として機能させる。

| 状況 | 通報先 |
|------|--------|
| 緊急 | 110番 |
| 騒音 | 自治体窓口 |
| 不法投棄 | 市役所リンク |
| 管理物件 | 管理会社案内 |

---

## 2. データベース設計

### users

| カラム | 型 | 備考 |
|--------|----|------|
| id | uuid | PK |
| created_at | timestamp | |
| status | enum | `active` / `banned` |
| trust_score | int | 初期値 100、スコアが低いと手動審査 |

※ 匿名可（メールのみ）

---

### incidents

| カラム | 型 | 備考 |
|--------|----|------|
| id | uuid | PK |
| mesh_code | string | 100m〜200m グリッドコード |
| category | enum | 下記カテゴリ参照 |
| description | text | 200文字制限 |
| time_range | enum | 時間帯（選択式） |
| image_url | string? | optional、処理済み画像のみ |
| status | enum | `pending` / `published` / `hidden` |
| created_at | timestamp | |
| published_at | timestamp? | 遅延公開後に設定 |
| user_id | uuid | FK → users |

**カテゴリ（category）**

| 値 | 表示名 |
|----|--------|
| `NOISE` | 騒音 |
| `DANGEROUS_BEHAVIOR` | 危険行為 |
| `ILLEGAL_DUMPING` | ゴミ不法投棄 |
| `ILLEGAL_PARKING` | 迷惑駐車 |
| `OTHER` | その他 |

**時間帯（time_range）**

| 値 | 時間帯 |
|----|--------|
| `MIDNIGHT` | 0:00〜6:00 |
| `MORNING` | 6:00〜10:00 |
| `DAYTIME` | 10:00〜16:00 |
| `EVENING` | 16:00〜20:00 |
| `NIGHT_EARLY` | 20:00〜22:00 |
| `NIGHT_LATE` | 22:00〜24:00 |

※ 緯度経度は保存しない（mesh_code のみ保存）

---

### reports（通報）

| カラム | 型 | 備考 |
|--------|----|------|
| id | uuid | PK |
| incident_id | uuid | FK → incidents |
| user_id | uuid | FK → users |
| reason | enum | 通報理由 |
| created_at | timestamp | |

**通報理由（reason）**

| 値 | 表示名 |
|----|--------|
| `PERSONAL_INFO` | 個人情報が含まれる |
| `FALSE_REPORT` | 虚偽・誇張の通報 |
| `HARASSMENT` | 嫌がらせ目的 |
| `OTHER` | その他 |

---

## 3. 権限設計

### 一般ユーザー

- 投稿
- 閲覧（遅延後）
- 通報

### モデレーター

- 公開承認
- 非公開化
- ユーザー BAN
- NGワード追加

### システム制御フロー

```
投稿 → status: pending
  ↓
AIフィルタ
  ├─ 通過 → 自動公開 (status: published, published_at: 設定)
  └─ 要審査 → モデレーター手動確認
```

---

## 4. モデレーション設計

### ① 自動フィルタ（NGワード）

以下を検出した場合、自動差し戻し：

- 人種・職業・外見のラベリング
- 「〇〇という人物」など個人特定表現
- 住所記載
- 車ナンバー形式（正規表現で検出）
- 電話番号形式（正規表現で検出）

### ② 画像処理

- 顔検出 → 強制モザイク
- ナンバープレート検出 → モザイク
- EXIF削除
- 原本を保存しない

### ③ スコアリング

| 条件 | アクション |
|------|-----------|
| 通報3件以上 | 自動非表示 |
| trust_score が低い | 手動審査必須 |

---

## 5. 炎上回避仕様

以下の機能は**すべて排除**する：

| 機能 | 理由 |
|------|------|
| リアルタイム表示 | 拡散抑止 |
| 住所ピン | 特定防止 |
| 人物属性の記述 | 個人特定防止 |
| コメント機能 | 炎上抑止 |
| いいね機能 | バズり防止 |
| ユーザーランキング | 競争排除 |
| SNS共有ボタン | 拡散防止 |

> "バズる設計"は全部切る。晒し要素を入れた瞬間に火薬庫になる。

---

## 6. ログと監査

- IPログ保持（一定期間）
- 端末IDハッシュ保存
- 法的開示請求対応フロー準備

---

## 7. 技術的注意事項（Native）

| 項目 | 内容 |
|------|------|
| 位置情報 | 取得後すぐメッシュ変換 → 生緯度を即時破棄 |
| バックグラウンド位置取得 | 禁止 |
| 端末内キャッシュ | 暗号化必須 |
| オフライン投稿 | サーバー側で必ず再検証 |
