# 近隣インシデントレポート 要件定義

## プロダクトコンセプト

> **「人を晒す地図」ではなく「地域の傾向を可視化する統計アプリ」**

- 個人を晒さない
- 「人」ではなく「出来事（インシデント）」を共有する
- 位置・時間はぼかす
- 通報導線を優先する

---

## 1. 画面設計

### ① マップ画面（Home）

**表示する情報**

- 100m〜200m グリッド単位のヒートマップ
- カテゴリフィルタ（画面上部にフィルターボタンを設置）
- 期間フィルタ（過去 24h / 7 日 / 30 日）

**表示しない情報**

- ピン（メッシュ単位でしかわからない、ピンは出さない）
- 正確な住所

---

### ② インシデント詳細画面

マップ画面上で丸をタップすると遷移できる

**表示する情報**

- カテゴリ
- 時間帯（例：22:00〜24:00）
- メッシュ位置（例：渋谷区 ○○ 100m グリッド）
- テキスト（200 文字以内）
- 同メッシュ件数（統計）

**表示しない情報**

- 人物特徴
- 車のナンバー
- 顔写真
- 個人特定可能な情報

---

### ③ 投稿画面

マップ画面の右下にあるプラスアイコンをタップすると遷移できる

**入力項目**

| 項目       | 必須 | 備考                                               |
| ---------- | :--: | -------------------------------------------------- |
| カテゴリ   |  ✅  | 騒音 / 危険行為 / ゴミ不法投棄 / 迷惑駐車 / その他 |
| 発生時間帯 |  ✅  | 選択式（時間帯を選ぶ）                             |
| 位置       |  ✅  | 自動メッシュ変換（生緯度は即時破棄）               |
| 内容       |  ✅  | 200 文字制限、NG ワードフィルタあり                |
| 画像       |  ❌  | 任意、自動モザイク処理必須                         |

**制限事項**

- 正確な位置は保存しない（mesh_code のみ保存）
- 同一ユーザーの連投制限（1 日 20 件まで）

---

## 2. 権限設計

### 一般ユーザー

- 投稿
- 閲覧（遅延後）
- 通報

### モデレーター

- 非公開化
- ユーザー BAN
- NG ワード追加

### システム制御フロー

```
投稿 → status: pending
  ↓
AIフィルタ
  ├─ 通過 → 自動公開 (status: published, published_at: 設定)
  └─ 要審査 → モデレーター手動確認
```

---

## 4. モデレーション設計

### ① 自動フィルタ（NG ワード）

以下を検出した場合、自動差し戻し：

- 人種・職業・外見のラベリング
- 「〇〇という人物」など個人特定表現
- 住所記載
- 車ナンバー形式（正規表現で検出）
- 電話番号形式（正規表現で検出）

### ② 画像処理

- 顔検出 → 強制モザイク
- ナンバープレート検出 → モザイク
- EXIF 削除
- 原本を保存しない

### ③ スコアリング

| 条件               | アクション   |
| ------------------ | ------------ |
| 通報 3 件以上      | 自動非表示   |
| trust_score が低い | 手動審査必須 |

---

## 4. 炎上回避仕様

以下の機能は**排除**する：

| 機能           | 理由         |
| -------------- | ------------ |
| 住所ピン       | 特定防止     |
| 人物属性の記述 | 個人特定防止 |

---

## 6. ログと監査

- IP ログ保持（一定期間）
- 端末 ID ハッシュ保存
- 法的開示請求対応フロー準備

---

## 7. 技術的注意事項（Native）

| 項目                     | 内容                                      |
| ------------------------ | ----------------------------------------- |
| 位置情報                 | 取得後すぐメッシュ変換 → 生緯度を即時破棄 |
| バックグラウンド位置取得 | 禁止                                      |
| 端末内キャッシュ         | 暗号化必須                                |
| オフライン投稿           | サーバー側で必ず再検証                    |
